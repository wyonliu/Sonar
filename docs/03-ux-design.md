# 03 · UX 设计规范

## 设计哲学

参照：Calm 的「做减法」+ Spotify 的「数据即美学」+ Headspace 的「引导而非灌输」

**四个设计原则**：
1. **情绪先于信息**：每个界面先触达情绪，再传递内容
2. **减少决策疲劳**：每个界面只有一个核心动作
3. **意外的准确感**：结果文字要让人感到被「看穿」
4. **天然分享欲**：最终输出物美到让人想主动传播

## 核心界面设计

### 界面 1：开场（电影级 30 秒）

```
屏幕完全黑暗
        ↓
渐显：一对人的轮廓，背景虚化的城市夜景
        ↓
无任何文字，只有轻柔的弦乐
        ↓
画面定格，轻微颗粒感，胶片质感
        ↓
从底部滑入一行字：
「你从未如此了解自己」
        ↓
1.5 秒后，出现：
「开始探索」按钮（极简，白色文字）
```

**设计要点**：
- 不要 Logo，不要 App 名字，不要功能说明
- 情绪先行，信息后置
- 30 秒内让用户感到「这个不一样」

---

### 界面 2：测试界面

```
顶部（10%）：进度条（极细，渐变色，从左到右填充）
              + 当前章节名「第一章：相识」

中间（75%）：
  [视频模式]
    全屏视频自动播放（无声，有字幕选项）
    视频下方：场景描述文字（1-2 行）

  [图像模式]
    2×2 网格，4 张图片
    无任何文字说明，纯视觉选择

底部（15%）：4 个选项
  每个选项：圆角矩形，半透明背景
  选择后：选中项高亮，0.3 秒后自动进入下一题
```

**微交互设计**：
- 选择后有轻微震动反馈（haptic）
- 进度条实时更新
- 每完成 5 题，短暂「已解锁一个维度」提示（0.5 秒）

---

### 界面 3：星图揭示（最重要的时刻）

参照：Spotify Wrapped 的揭示体验——逐步揭开，每一步都有惊喜

```
Step 1（2 秒）：屏幕全黑，一句话从中心出现：
「你的声呐信号已经返回」

Step 2（3 秒）：7 条射线从中心向外延伸，形成框架
每条线延伸时有轻微音效（类似声呐的 ping 声）

Step 3（3 秒）：多边形从外向内填充，带颜色渐变
同时显示 7 个维度标签

Step 4（持续显示）：完整星图 + 三行核心洞察文字
「你在关系里是一个____的人」
「你最大的力量是____」
「你最深的渴望是____」

Step 5：向上滑动，进入「关系阴影」（付费后解锁完整版）
```

### 界面 4：关系阴影（核心转化页）

```
背景：深色，略微模糊的星图作为底纹

标题：「你不知道的自己」

正文（免费部分，前 2 句话）：
「你以为自己是____。
 但你的 23 个选择显示，真相是……」
                    ↑
             [模糊遮罩，渐变]

解锁按钮：
「解锁完整洞察  ¥9.9」
副文字：「87% 的用户说：准到可怕」

底部：
「已有 312,847 人发现了关系里真实的自己」
```

**设计逻辑**：
- 让用户看到前 2 句话，已经有感觉了
- 模糊遮罩制造「欲看未看」的张力
- 社交证明（87% / 30 万人）降低付费门槛

### 界面 5：关系故事 + 分享

```
「你的关系故事」

[AI 生成的 200 字叙述]
以第三人称，描述「这个人在感情里是什么样的人」

底部操作：
[分享星图]  [查看你的匹配]

分享卡片设计：
- 星图多边形（主视觉）
- 3 行关键描述
- Sonar 品牌标识（极小）
- 适配 朋友圈/小红书/微博
```

## 声音设计

| 场景 | 声音设计 |
|------|---------|
| 开场 | 轻柔弦乐，20 秒，无歌词 |
| 测试中 | 极低底噪，轻微环境音 |
| 选择题目 | 轻微 haptic，无声音 |
| 完成章节 | 一声轻柔的「叮」，类似冥想铃 |
| 星图生成 | 声呐 ping 音效序列，7 声，对应 7 维度 |
| 解锁内容 | 轻微展开音效 |

## 关键体验细节

### 不要让用户等待

- 视频预加载（当前题目播放时，预加载下 3 题）
- 分析在测试过程中后台实时计算
- 星图生成时间 < 2 秒（视觉上做 3 秒动画，但实际已经计算完）

### 保护用户的情绪

- 测试中不显示「剩余题目数」（避免让用户急着完成）
- 只显示章节进度（第 1 章 / 第 2 章 / 第 3 章）
- 可以在任何时候暂停，下次从断点继续

### 分享的细节

- 分享卡片不要有「扫码下载」（太功利）
- 只有 Sonar · 声呐 的小 Logo
- 让星图本身成为主角，不是 App 的广告

## 响应式设计

**移动端优先**，三种形态：

| 形态 | 适用场景 | 特点 |
|------|---------|------|
| iOS/Android App | 完整体验 | 视频、haptic、原生动画 |
| 微信小程序 | 分享落地页 | 无视频，图片场景替代 |
| Web | 桌面用户 | 完整功能，鼠标优化 |
