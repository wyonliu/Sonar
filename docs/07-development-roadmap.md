# 07 · 开发路线图

## MVP 定义

> MVP 的唯一目标：让 50 个真实用户做完测试，其中 40+ 人说「准到可怕」。

在此之前，没有匹配功能，没有付费功能，没有任何其他功能。

---

## 里程碑

### M0 · 基础设施（第 1-2 周）

**目标**：把环境搭起来，跑通最小链路

```
□ GitHub 仓库初始化（当前步骤）
□ 本地开发环境
  □ FastAPI 后端启动
  □ Next.js Web 启动
  □ React Native App 启动
  □ PostgreSQL + Redis 本地实例
□ 基础 Auth（手机号 + 验证码）
□ CI/CD 管道（GitHub Actions → Railway）
□ Supabase 数据库初始化（核心表结构）
```

### M2 · 内测与迭代（第 7-10 周）

**目标**：50 个真实用户，准确率 > 80%

```
□ 内测招募（微信朋友圈，非公开）
  □ 第一批：10 人（身边朋友，给最深度反馈）
  □ 第二批：20 人（朋友的朋友）
  □ 第三批：20 人（更陌生的人，测试泛化能力）
□ 每次内测后
  □ 深度访谈：「哪道题让你感觉最准？哪道最不准？」
  □ 量化评分：1-5 分，准确度
  □ 根据反馈迭代题目内容
□ 迭代重点
  □ 让「关系阴影」的文字更准更有穿透力
  □ 让星图更美，让人想分享
  □ 优化测试节奏（不能让人觉得累）
```

**M2 完成标准**：50 人内测，平均准确度评分 > 4.0/5.0，自发分享率 > 50%

## 优先级原则

**永远最高优先级**：测试的准确性和「准到可怕」的感受

**不要过早做的事**：
- 视频场景（M3 之后再做，先用高质量插画）
- 语音分析（M4 之后）
- 复杂的 AI coaching（M4 之后）
- 企业版/职场版（M6 之后）

**快速迭代的心态**：
- 题目内容随时可以修改，不是代码，成本低
- 分析算法随时可以优化
- 视觉随时可以打磨
- 但基础架构要一次做对

## 团队配置建议

| 角色 | 职责 | 优先级 |
|------|------|--------|
| 关系心理学顾问 | 测试题目设计、分析框架校验 | **最高**，没有这个人不要启动 |
| 全栈工程师 × 2 | 后端 API + 前端 App | 高 |
| AI 工程师 × 1 | Claude 集成、匹配算法 | 高 |
| 视觉设计师 × 1 | 星图、分享卡片、整体视觉 | 高 |
| 产品经理 × 1 | 需求、测试、用户反馈 | 中 |

**第一阶段（M0-M2）最小团队**：
- 1 个心理学顾问（兼职/顾问形式）
- 2 个工程师
- 1 个设计师
