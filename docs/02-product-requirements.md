## 核心功能设计

### F1 · 声呐测试（核心产品）

**设计原则**：测试本身就是产品，不是通往产品的门槛。

参照：Duolingo 的第一课就是学会说一句话，立刻给到成就感。

#### 测试结构

```
第一章：相识期（约 10 分钟）
├── 视频场景 × 5（亲密节奏、主动性、边界感）
├── 图像投射 × 3（原型/潜意识偏好）
└── 语音片段 × 1（可选，AI 分析叙事模式）

第二章：磨合期（约 8 分钟）
├── 视频场景 × 4（冲突模式、沟通风格）
├── 选择情境 × 4（承诺意愿、亲密需求）
└── 八字维度（输入生日，自动生成）

第三章：深水期（约 7 分钟）
├── 视频场景 × 3（压力下的真实反应）
├── 价值观情境 × 5（金钱、家庭、事业、自由、亲密）
└── 原型投射 × 2（塔罗意象选择）
```

#### 场景题设计规范

**好题的标准**：让用户在情绪里做选择，而不是在理性里思考答案。

```
❌ 错误示范：
"你在感情里遇到冲突时，你会主动沟通还是需要冷静一下？"
→ 用户会填理想答案

✅ 正确示范：
[视频：你和 TA 因为一件小事冷战了两天。今天早上你先醒来，
看着还在睡觉的 TA……]
A. 轻轻把被子给 TA 盖好，什么都不说
B. 把 TA 叫醒，说"咱们谈谈吧"
C. 悄悄起床，看看 TA 有没有主动联系
D. 发一条消息："我想好了，你说得对"
→ 选择揭示真实行为模式
```

每道题同时测量 3-5 个心理维度，不是一题一维度。

### F2 · 关系星图（核心输出）

参照：Spotify Wrapped——每年最期待的个人数据时刻，人们主动分享。

#### 星图构成

7 个维度，每个维度 0-100 分，形成唯一的多边形图案：

| 维度 | 测量内容 |
|------|---------|
| 亲密节奏 | 你建立亲密感需要多长时间 |
| 表达强度 | 你在关系里的情感表达程度 |
| 自主需求 | 你需要的个人空间大小 |
| 冲突风格 | 你处理冲突的方式 |
| 安全基线 | 你在关系里的基础安全感水平 |
| 承诺深度 | 你进入深度承诺的意愿和速度 |
| 成长取向 | 你在关系里是否重视共同成长 |

#### 星图视觉规范

- 7 轴多边形，动态生成
- 颜色系统：
  - 暖色系（金/橙）= 高表达、高亲密需求
  - 冷色系（蓝/紫）= 高独立、高内敛
  - 绿色系 = 高安全感、高成长取向
- 分享尺寸：1080×1080，适配朋友圈/小红书
- 含简短文字标签，无需解释即可理解

### F3 · 关系阴影揭示（差异化核心）

参照：Headspace 的「你不是你的想法」——挑战用户对自己的固有认知。

**逻辑**：

```python
self_perception = derive_from_stated_preferences()   # 用户认为自己是什么样
actual_behavior = derive_from_scenario_choices()     # 场景里的实际选择
shadow = calculate_divergence(self_perception, actual_behavior)

if shadow.is_significant():
    output = f"""
    你认为自己是「{self_perception.label}」，
    但你的 23 个选择显示，你在关系里实际上更接近「{actual_behavior.label}」。

    这个落差通常意味着：{shadow.interpretation}

    这不是你的弱点——这是你还没有被真正看见的部分。
    """
```

这段话是用户最愿意截图分享的内容。设计目标：**让 70% 的用户看到后说「准到可怕」**。

### F4 · 匹配发现（变现核心）

#### 匹配逻辑

不是「找和你相似的人」，而是「找和你相容的人」。

互补性 > 相似性（Gottman 研究支持）：

```
你的冲突风格：回避型  ←→  最佳匹配：温和追近型（非激进型）
你的亲密节奏：慢热   ←→  最佳匹配：耐心型，但有稳定信号
你的表达强度：低     ←→  最佳匹配：善于解读非语言信号的人
```

#### 匹配展示流程

```
Step 1：看到匹配数量（模糊）
"已有 23 人与你的关系星图高度共鸣"

Step 2：付费解锁第一个人的「关系故事」（无照片）
AI 生成的 200 字介绍：这个人在关系里是什么样的人

Step 3：双方都表达意向后，才互相看到照片

Step 4：AI 生成「你们的共鸣图」
→ 你们在哪些维度高度共鸣，哪些维度形成张力，以及如何相处
```

**照片最后出现**——这是 Sonar 最重要的设计决策之一。

### F5 · AI 关系故事（替你说话）

这是解决「难被第一眼发现」问题的核心功能。

参照：Hinge 的 Prompts——让人格在见面前先被感受到。

**生成逻辑**：

```
输入：用户的关系星图 + 场景选择数据 + 语音分析
输出：一段 200-300 字的个性化叙述

示例输出：
"他是一个你需要时间才能真正认识的人——但值得。
他不会在第一次见面就把自己全部展示出来，
但当他决定让你进来，他会让你感到一种少见的稳定感。
他记得你说过的每一件小事。他不善于说「我爱你」，
但他会在你需要之前，就把你需要的东西放到你面前。
他有自己的宇宙，写科幻，思考人性——
但他不会让那个宇宙把你排除在外。"
```

### F6 · 双人共鸣模式

参照：Couple App 的共同体验设计。

**触发**：分享星图 → 对方扫码或点击 → 自动进入双人比较

**输出**：
- 共鸣图（两个星图叠加的可视化）
- 共鸣分析：你们天然和谐的 3 个维度
- 张力分析：你们容易产生误解的 2 个维度（以及原因）
- 相处建议：基于你们的组合，AI 给出的具体建议

**设计关键**：张力分析要让人看了说「原来是这样」而不是「你们不合适」。关系里的差异本身不是问题，不理解差异才是问题。

## 用户旅程地图

```
[发现]          朋友圈/小红书看到精美星图
                        ↓
[入场]          30 秒电影开场，无文字，情绪先行
                        ↓
[沉浸]          章节式测试，每题都有情绪反馈
                        ↓
[高峰时刻]      星图生成 + 关系阴影揭示
                "这也太准了" → 截图分享
                        ↓
[钩子]          "已有 23 人与你共鸣"（模糊）
                        ↓
[转化]          ¥9.9 解锁完整分析
                ¥29 解锁第一个关系故事
                        ↓
[深度]          继续匹配 → 付费订阅
                邀请伴侣做双人模式
                        ↓
[传播]          分享星图 → 新用户进入
```
